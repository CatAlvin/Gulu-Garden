# 《咕噜小菜园》Gulu Garden 版本路线文档 v0.1

## 1. 版本规划原则

《咕噜小菜园》是一个长期单人开发项目，因此版本规划必须遵守以下原则：

1. 每个版本只解决一个主要问题。
2. 每个版本都要能运行和测试。
3. 不为了未来功能牺牲当前可玩性。
4. 复杂功能必须后置。
5. 每个版本完成后都要更新 README 和开发日志。
6. 项目优先保证稳定，而不是堆功能。

---

## 2. 总体路线

项目路线分为五个阶段：

| 阶段      | 目标                   |
| ------- | -------------------- |
| Phase 1 | 做出最小可玩种菜闭环           |
| Phase 2 | 加入存档、离线成长和昼夜系统       |
| Phase 3 | 加入任务、图鉴和更多作物         |
| Phase 4 | 加入品质、知识锁、装饰和音效动画     |
| Phase 5 | 加入 AI 宠物、好友农场等长期扩展内容 |

---

# Phase 1：基础可玩闭环

## Version 0.1 — 项目骨架与基础窗口

### 目标

建立工程化项目结构，并让 Pygame 窗口成功运行。

### 功能范围

* 创建项目目录结构
* 创建 `src/`、`assets/`、`data/`、`saves/`、`docs/`
* 创建 Pygame 窗口
* 设置窗口大小为 1280 × 720
* 设置基础 FPS
* 支持正常关闭窗口
* 显示基础背景色或占位背景图

### 不做内容

* 不做农田
* 不做作物
* 不做商店
* 不做存档
* 不做动画

### 验收标准

* 运行项目后可以打开窗口
* 关闭窗口不会报错
* 项目结构清晰
* README 中说明如何运行项目

---

## Version 0.2 — 农田网格与鼠标点击

### 目标

让玩家看到土地，并能点击土地。

### 功能范围

* 显示 8 块土地
* 初始开放 4 块土地
* 其余土地显示为 locked 状态
* 支持鼠标点击土地
* 点击土地后显示土地编号和状态
* 土地状态至少包括 empty 和 locked

### 不做内容

* 不做作物成长
* 不做播种
* 不做商店
* 不做金币变化

### 验收标准

* 玩家能看到土地格子
* 玩家点击不同土地时，系统能识别点击的是哪一块
* locked 土地不能被操作
* empty 土地能显示“可以播种”的提示

---

## Version 0.3 — 星泡萝卜播种与成长

### 目标

实现最小作物成长系统。

### 功能范围

* 添加星泡萝卜基础数据
* 支持选择星泡萝卜种子
* 点击空地可以播种
* 土地状态从 empty 变为 planted / growing
* 作物根据时间成长
* 成熟后状态变为 mature
* UI 显示作物状态

### 不做内容

* 不做商店购买
* 不做金币消耗
* 不做存档
* 不做离线成长
* 不做任务

### 验收标准

* 玩家可以在空地种下星泡萝卜
* 作物能随着时间变为成熟
* 成熟状态可以被识别
* 不同土地的作物状态互不影响

---

## Version 0.4 — 收获与金币循环

### 目标

让玩家可以收获作物并获得金币。

### 功能范围

* 增加玩家金币
* 初始金币为 50
* 成熟作物可以被点击收获
* 收获后土地恢复 empty
* 收获星泡萝卜获得 20 金币
* 顶部 UI 显示金币数量

### 不做内容

* 不做商店
* 不做背包
* 不做存档
* 不做任务

### 验收标准

* 玩家收获成熟星泡萝卜后，金币增加
* 土地恢复为空地
* 未成熟作物不能收获
* locked 土地不能收获

---

## Version 0.5 — 商店与种子背包

### 目标

完成买种子、种菜、收获、赚钱的基础经济闭环。

### 功能范围

* 增加商店按钮
* 商店出售星泡萝卜种子
* 星泡萝卜种子价格为 10 金币
* 购买种子消耗金币
* 背包记录星泡萝卜种子数量
* 播种时消耗 1 个种子
* 金币不足时不能购买
* 种子数量不足时不能播种

### 不做内容

* 不做完整背包界面
* 不做多种商品
* 不做肥料
* 不做工具
* 不做图鉴

### 验收标准

* 玩家必须购买种子才能播种
* 播种会减少种子数量
* 收获会增加金币
* 玩家可以通过种植反复赚钱

---

# Phase 2：存档、离线成长与昼夜

## Version 0.6 — 单存档系统

### 目标

让游戏可以保存和读取当前进度。

### 功能范围

* 保存金币
* 保存种子数量
* 保存每块土地状态
* 保存作物类型
* 保存作物种植时间
* 保存作物是否浇水
* 保存最后保存时间
* 启动游戏时读取存档

### 不做内容

* 不做多个存档槽
* 不做存档选择界面
* 不做防作弊
* 不做云存档

### 验收标准

* 玩家种下作物后关闭游戏
* 重新打开游戏后，土地和金币状态仍然正确
* 存档文件可以被人工查看
* 存档损坏时游戏不会直接崩溃

---

## Version 0.7 — 离线成长

### 目标

让关闭游戏期间时间继续流逝。

### 功能范围

* 记录作物种植时间
* 读取存档时计算当前时间与种植时间差
* 根据经过时间更新作物状态
* 如果经过时间超过成熟时间，则作物变为 mature
* UI 提示玩家离线期间发生了什么

### 不做内容

* 不做复杂离线收益总结
* 不做作物枯萎
* 不做系统时间作弊检测

### 验收标准

* 玩家种下 30 秒成熟的星泡萝卜
* 关闭游戏超过 30 秒
* 再次打开游戏时，作物已经成熟

---

## Version 0.8 — 多存档系统

### 目标

支持多个玩家进度。

### 功能范围

* 支持至少 3 个存档槽
* 可以新建存档
* 可以读取指定存档
* 可以覆盖保存
* 存档列表显示基本信息，例如金币、游戏时间、最后保存时间

### 不做内容

* 不做云同步
* 不做存档缩略图
* 不做存档加密

### 验收标准

* 三个存档互不影响
* 删除或损坏一个存档不影响其他存档
* 玩家能明确知道自己进入的是哪个存档

---

## Version 0.9 — 昼夜系统基础版

### 目标

根据现实时间显示清晨、白日、傍晚和午夜。

### 功能范围

* 根据系统时间判断当前阶段
* 顶部 UI 显示当前阶段
* 不同阶段使用不同背景色调或背景图
* 阶段包括 Morning、Daytime、Evening、Midnight

### 不做内容

* 不做昼夜平滑渐变
* 不做作物加成
* 不做天气
* 不做季节

### 验收标准

* 不同现实时间进入游戏时，显示正确昼夜阶段
* 玩家能明显感受到背景变化

---

# Phase 3：任务、图鉴与内容扩展

## Version 1.0 — 第一个可试玩版本

### 目标

整合基础功能，形成可以展示和试玩的版本。

### 功能范围

* Pygame 窗口
* 土地点击
* 星泡萝卜种植
* 作物成长
* 收获金币
* 商店购买种子
* 种子背包
* 单存档或多存档
* 离线成长
* 昼夜显示
* 基础 UI
* README 运行说明

### 验收标准

* 可以发给朋友试玩
* 可以录屏展示
* 可以上传 GitHub
* 玩家能理解基础玩法
* 游戏关闭再打开后进度正确

---

## Version 1.1 — 简单任务系统

### 目标

加入短期目标，让玩家有明确任务。

### 功能范围

* 添加任务数据文件
* 添加任务栏
* 记录任务进度
* 完成任务后领取奖励
* 第一批任务围绕星泡萝卜

### 示例任务

| 任务名   | 目标         |    奖励 |
| ----- | ---------- | ----: |
| 第一次播种 | 种下 1 个星泡萝卜 | 10 金币 |
| 第一次收获 | 收获 1 个星泡萝卜 | 20 金币 |
| 小菜园起步 | 收获 3 个星泡萝卜 | 50 金币 |

### 验收标准

* 玩家完成任务后能领取奖励
* 任务进度能保存
* 重启游戏后任务进度不丢失

---

## Version 1.2 — 图鉴基础版

### 目标

记录玩家种植和收获过的作物。

### 功能范围

* 添加图鉴入口
* 显示星泡萝卜基础信息
* 记录是否种植过
* 记录是否收获过
* 最佳种植方式显示“未解锁”

### 不做内容

* 不做完美品质
* 不做知识锁解锁
* 不做复杂图鉴动画

### 验收标准

* 玩家收获星泡萝卜后，图鉴显示已收获
* 图鉴信息能保存

---

## Version 1.3 — 多作物基础版

### 目标

加入更多奇幻作物，验证数据驱动扩展能力。

### 功能范围

* 新增 2 到 3 种作物
* 商店显示多种种子
* 不同作物拥有不同价格、成熟时间和出售价格
* 图鉴支持多种作物

### 验收标准

* 新增作物主要通过数据文件配置
* 不需要大规模修改核心逻辑
* 每种作物能独立成长和收获

---

# Phase 4：特色系统

## Version 2.0 — 完美品质系统

### 目标

加入作物品质，让收获有惊喜感。

### 功能范围

* 每次收获有概率获得完美品质
* 完美品质可进入图鉴收藏
* 普通品质和完美品质有不同提示
* 完美品质可以触发额外奖励

### 验收标准

* 玩家收获时有概率获得完美品质
* 完美品质记录能保存
* 图鉴能显示是否获得过完美品质

---

## Version 2.1 — 知识锁图鉴

### 目标

让图鉴成为探索系统，而不是普通资料页。

### 功能范围

* 每种作物拥有隐藏最佳种植方式
* 获得完美品质后解锁对应描述
* 图鉴显示已解锁知识
* 未解锁前只显示模糊提示

### 验收标准

* 未获得完美品质前，最佳种植方式不可见
* 获得完美品质后，最佳种植方式永久解锁
* 解锁状态能保存

---

## Version 2.2 — 昼夜影响作物

### 目标

让昼夜系统真正影响玩法。

### 功能范围

* 每种作物可以设置偏好时间
* 偏好时间影响成长速度或完美品质概率
* 图鉴中可以逐步揭示这些偏好

### 验收标准

* 同一作物在不同时间种植会有不同效果
* 效果不能过于复杂，玩家应能理解

---

## Version 2.3 — 基础动画与音效

### 目标

提升游戏反馈和观感。

### 功能范围

* 点击音效
* 播种音效
* 浇水音效
* 收获音效
* 金币弹出动画
* 作物成熟提示动画
* 星泡萝卜成熟时星星泡泡特效

### 验收标准

* 玩家操作时有清晰反馈
* 动画不影响核心逻辑
* 没有音频文件时游戏仍能运行

---

# Phase 5：长期扩展内容

## Version 3.0 — 装饰系统

### 目标

让玩家可以个性化自己的小菜园。

### 功能范围

* 装饰商店
* 装饰背包
* 简单摆放系统
* 装饰保存

### 验收标准

* 玩家可以购买并摆放装饰
* 重启游戏后装饰位置仍然存在

---

## Version 4.0 — AI 聊天宠物

### 目标

加入游戏内陪伴宠物，提升情感体验。

### 功能范围

* 宠物显示在界面中
* 宠物能根据当前游戏状态说话
* prompt 包含金币、作物状态、任务状态和图鉴状态
* 接入 DeepSeek API 或其他模型 API

### 注意事项

该功能需要：

* API Key 管理
* 网络请求
* 请求频率限制
* 错误处理
* 离线模式处理
* 成本控制

因此不适合早期开发。

---

## Version 5.0 — 好友农场或模拟好友

### 目标

加入社交感。

### 可能方向

1. 模拟好友农场
   不联网，只生成几个 NPC 好友菜园。

2. 真好友系统
   需要服务器、账号、数据库和网络通信。

### 推荐路线

优先做模拟好友农场，不建议早期做真实联网好友系统。

---

# 当前推荐推进顺序

当前最推荐的开发顺序是：

1. Version 0.1：项目骨架与窗口
2. Version 0.2：农田网格与点击
3. Version 0.3：星泡萝卜播种与成长
4. Version 0.4：收获与金币
5. Version 0.5：商店与种子背包
6. Version 0.6：单存档
7. Version 0.7：离线成长
8. Version 0.9：昼夜系统
9. Version 1.0：第一个可试玩版本

多存档可以根据实际开发难度放到 Version 1.0 前后。
