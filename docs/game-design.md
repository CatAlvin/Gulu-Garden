# 《咕噜小菜园》Gulu Garden 游戏设计文档 v0.1

## 1. 游戏概述

《咕噜小菜园》（Gulu Garden）是一款使用 Python 与 Pygame 开发的可爱休闲种菜经营游戏。

玩家在自己的小菜园中购买种子、播种、浇水、等待作物成长、收获并卖出作物，从而获得金币。金币可以继续用于购买种子、解锁土地、升级工具、购买装饰，并逐步扩展自己的菜园。

游戏的长期特色是：

1. 奇幻作物
2. 现实时间流逝
3. 离线成长
4. 昼夜变化
5. 知识锁图鉴
6. 完美品质收集
7. 可爱休闲的陪伴感

本项目是一个长期单人开发项目，因此第一阶段不追求完整系统，而是先实现最小可玩闭环。

---

## 2. 游戏定位

### 2.1 类型定位

本游戏主要属于：

* 休闲种菜游戏
* 放置经营游戏
* 单机 Pygame 作品集项目
* 可长期扩展的个人游戏开发项目

它的玩法接近 QQ 农场的核心循环：

> 买种子 → 播种 → 等待成长 → 收获 → 卖钱 → 继续扩展农场

同时也具有放置挂机游戏的特点：

> 即使玩家关闭游戏，作物也会根据现实经过的时间继续成长。

---

## 3. 目标玩家体验

玩家打开游戏时，应获得一种轻松、可爱、舒服的感觉。

目标体验包括：

1. 操作简单
   玩家不需要复杂键位，主要通过鼠标点击完成操作。

2. 反馈清晰
   玩家点击土地、购买种子、播种、浇水、收获时，都应该看到明确反馈。

3. 有持续目标
   玩家不仅是重复收菜，还可以通过任务、图鉴、作物收集和土地解锁获得成长感。

4. 有发现感
   每种奇幻作物都有隐藏的最佳种植方式。玩家通过实验、收集和图鉴解锁逐渐了解作物特性。

5. 有陪伴感
   后期可以加入游戏内聊天宠物，使玩家感觉小菜园是一个会回应自己的空间。

---

## 4. 核心玩法循环

### 4.1 最小核心循环

第一阶段只实现以下循环：

1. 玩家进入游戏。
2. 玩家查看金币数量。
3. 玩家打开商店。
4. 玩家购买星泡萝卜种子。
5. 玩家选择星泡萝卜种子。
6. 玩家点击空地播种。
7. 玩家给作物浇水。
8. 作物根据现实时间成长。
9. 作物成熟。
10. 玩家点击成熟作物收获。
11. 玩家获得金币。
12. 玩家继续购买种子或扩展土地。

这个循环是整个项目最重要的基础。

只要这个循环稳定，后续所有高级系统都可以围绕它扩展。

---

## 5. 作物设计

### 5.1 作物总体设定

本游戏中的作物不是普通现实蔬菜，而是奇幻作物。

奇幻作物应该具有以下特点：

* 名字有记忆点
* 外观可爱
* 有特殊成长习性
* 可以进入图鉴
* 有隐藏最佳种植方式
* 后期可以拥有完美品质
* 后期可以拥有成熟动画或特效

---

### 5.2 初始作物：星泡萝卜

| 项目     | 内容                    |
| ------ | --------------------- |
| 中文名    | 星泡萝卜                  |
| 英文名    | Starbubble Radish     |
| 定位     | 新手作物                  |
| 种子价格   | 10 金币                 |
| 成熟时间   | 30 秒，开发测试用            |
| 出售价格   | 20 金币                 |
| 成长阶段   | 种子、幼苗、生长期、成熟          |
| 是否需要浇水 | 是                     |
| 是否可能枯萎 | 后续版本加入                |
| 完美品质   | 后续版本加入                |
| 图鉴知识锁  | 后续版本加入                |
| 特点     | 成熟时像萝卜一样鼓起，周围可能冒出星星泡泡 |

### 5.3 星泡萝卜描述

星泡萝卜是一种适合新手种植的奇幻作物。它的根部会储存微弱星光，成熟时会从叶片之间冒出小小的星星泡泡。虽然它并不昂贵，但非常稳定，是玩家进入咕噜小菜园后的第一种作物。

---

## 6. 土地设计

### 6.1 土地定位

土地是作物生长的载体。

每一块土地都需要保存自己的状态，例如是否为空、是否已播种、作物是否成熟、是否需要浇水等。

### 6.2 土地状态

| 状态         | 说明       | 第一阶段是否实现 |
| ---------- | -------- | -------- |
| locked     | 被锁定，不能种植 | 是        |
| empty      | 空地，可以播种  | 是        |
| planted    | 已播种      | 是        |
| growing    | 生长中      | 是        |
| need_water | 需要浇水     | 是        |
| mature     | 已成熟，可以收获 | 是        |
| withered   | 枯萎       | 后续版本     |

### 6.3 第一阶段土地数量

第一阶段建议：

* 总土地数量：8 块
* 初始开放土地：4 块
* 锁定土地：4 块

理由：

1. 4 块开放土地足够测试基础玩法。
2. 8 块总土地可以提前展示“以后能扩展”的感觉。
3. 不会让第一版 UI 过于复杂。

---

## 7. 金币与经济系统

### 7.1 金币作用

金币是游戏初期最重要的资源。

金币可以用于：

* 购买种子
* 解锁土地
* 购买肥料
* 购买水壶
* 升级农田
* 购买装饰

### 7.2 第一阶段经济规则

第一阶段只实现最简单的经济循环：

> 买种子消耗金币，收获作物获得金币。

暂时不实现复杂市场、价格波动、税收、维护费等系统。

### 7.3 第一阶段数值

| 项目       | 数值 |
| -------- | -: |
| 初始金币     | 50 |
| 星泡萝卜种子价格 | 10 |
| 星泡萝卜出售价格 | 20 |
| 单次净利润    | 10 |
| 初始开放土地   |  4 |
| 总土地数量    |  8 |

---

## 8. 背包系统

### 8.1 背包长期目标

背包最终应支持：

* 种子
* 作物
* 工具
* 肥料
* 装饰
* 特殊道具

### 8.2 第一阶段背包范围

第一阶段只记录种子数量。

例如：

| 物品     | 数量 |
| ------ | -: |
| 星泡萝卜种子 |  3 |

第一阶段收获作物后，可以先直接转化为金币，不强制进入作物背包。

后续如果要做完整“作物背包 + 手动出售”，可以在之后版本加入。

---

## 9. 商店系统

### 9.1 商店定位

商店用于购买种子、工具、肥料和装饰。

### 9.2 第一阶段商店

第一阶段商店只出售：

| 商品     | 类型 |    价格 |
| ------ | -- | ----: |
| 星泡萝卜种子 | 种子 | 10 金币 |

### 9.3 商店交互

玩家打开商店后，应看到商品图标、名称、价格和购买按钮。

如果金币不足，应显示提示：

> 金币不够，暂时买不起哦。

如果购买成功，应显示提示：

> 获得 1 个星泡萝卜种子。

---

## 10. 时间与昼夜系统

### 10.1 现实时间流逝

游戏时间与现实时间同步。

作物成长不依赖玩家是否在线。

例如：

1. 玩家在 10:00 种下星泡萝卜。
2. 星泡萝卜需要 30 秒成熟。
3. 玩家 10:05 关闭游戏。
4. 玩家 10:10 重新打开游戏。
5. 星泡萝卜应该已经成熟。

### 10.2 昼夜阶段

游戏根据现实时间划分阶段。

| 时间范围          | 阶段 | 英文名      |
| ------------- | -- | -------- |
| 05:00 - 08:59 | 清晨 | Morning  |
| 09:00 - 16:59 | 白日 | Daytime  |
| 17:00 - 20:59 | 傍晚 | Evening  |
| 21:00 - 04:59 | 午夜 | Midnight |

### 10.3 第一阶段处理方式

第一阶段可以先显示当前阶段文字，例如：

> 当前时间：傍晚

后续版本再让背景、光照、作物成长速度和完美品质概率受到昼夜影响。

---

## 11. 任务系统

### 11.1 任务定位

任务系统给玩家提供短期目标，避免玩法变成单纯重复收菜。

### 11.2 第一批任务

| 任务名   | 目标         |    奖励 |
| ----- | ---------- | ----: |
| 第一次播种 | 种下 1 个星泡萝卜 | 10 金币 |
| 第一次收获 | 收获 1 个星泡萝卜 | 20 金币 |
| 小菜园起步 | 收获 3 个星泡萝卜 | 50 金币 |

### 11.3 任务后续扩展

未来可以扩展：

* 每日任务
* 主线任务
* 图鉴任务
* 成就任务
* 特殊事件任务

---

## 12. 图鉴系统

### 12.1 图鉴定位

图鉴是本游戏的重要特色系统。

它不是普通资料页，而是“知识锁”玩法的核心载体。

### 12.2 第一阶段图鉴

第一阶段可以暂时不实现图鉴界面，但需要在设计中预留。

如果实现基础图鉴，只需要显示：

* 作物名称
* 作物描述
* 是否种植过
* 是否收获过
* 最佳种植方式：未解锁

### 12.3 后期知识锁机制

后续加入完美品质后：

1. 玩家种植作物。
2. 玩家收获作物。
3. 每次收获有概率获得完美品质。
4. 第一次获得完美品质后，该作物进入图鉴收藏。
5. 图鉴解锁该作物的最佳种植方式描述。

例如：

> 星泡萝卜更喜欢傍晚播种。如果成熟前浇水一次，它更容易形成完美品质。

## 13. 用户界面设计

### 13.1 主界面信息

主界面应显示：

* 金币
* 当前选中的种子
* 当前时间
* 当前昼夜阶段
* 商店按钮
* 背包按钮
* 图鉴按钮
* 任务栏
* 作物状态提示
* 操作提示

### 13.2 第一阶段界面布局

建议 1280 × 720 初始布局：

```text
顶部：金币 / 当前时间 / 当前昼夜阶段
左侧：任务栏
中间：农田区域
底部：商店 / 背包 / 图鉴 / 当前选择物品
右侧：作物状态或操作提示
```

### 13.3 交互原则

第一阶段坚持简单交互：

* 鼠标点击土地
* 鼠标点击按钮
* 不做角色移动
* 不做复杂拖拽
* 不做多层菜单
* 点击后必须有提示反馈

## 16. 当前设计结论

《咕噜小菜园》的第一目标不是一次性做出完整农场，而是先做出一个稳定、可玩、可保存、可离线成长的基础种菜闭环。

第一阶段成功标准：

> 玩家可以买星泡萝卜种子，把它种到土地里，浇水，等待现实时间流逝，成熟后收获并获得金币。关闭游戏后再次打开，作物状态仍然正确。

只要这个基础循环完成，后续就可以逐步加入任务、图鉴、完美品质、装饰、AI 宠物和好友系统。
